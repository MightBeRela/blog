Github Pages x Hugo の運用方法

Github PagesやHugoの要所要所の使い方は多く存在するのですが、
Gitのインストールから実際にHugoを使ってブログとして使うまで
書いているところは少ないので運用方法として書くことにしました。

Hugoに関しては以下の記事を日本語訳したものを1つの記事にまとめたもとの考えてもらえば良いと思います。
https://gohugo.io/getting-started/quick-start/
https://gohugo.io/hosting-and-deployment/hosting-on-github/

GitHubアカウントの作成

  1. Githubの公式サイトにアクセス
  2. ユーザー名・メールアドレス・パスワードを入力
  3. Enforcement Challengesを認証する (画像を正しい向きにする)
  4. Join A FreePlanをクリック
  5. Conplete setupをクリック
  6. 上で入力したメールアドレスに送られてくるメールに記載されているVerify email addressをクリックする

リポジトリの作成

  1. Githubの管理ページ(https://github.com/ユーザーID)にアクセス
  2. Newをクリック
  3. Repository nameに好きな名前を入力
  4. Create repositoryをクリック

Gitをインストール(Windows)

  1. Git for Windowsの公式サイトにアクセス
  2. Downloadをクリック
  3. Git-X.XX.X-XX-bit.exeを起動
  4. すべてNextをクリック

Gitを設定する

  1. コマンドプロンプトを開く
  2. git config --global user.name ユーザーIDを実行
  3. git config --global user.email メールアドレスを実行 　　

Hugoをインストール(Windows)

  1. Hugoの公式リポジトリにアクセス
  2. hugo_X.XX.X_Windows-XXbit.zipをダウンロード
  3. 好きな場所に解凍

環境変数にHugoを追加する

  1. コントロールパネルを開く
  2. システムとセキュリティを開く
  3. システムを開く
  4. システムの詳細設定を開く
  5. 環境変数を開く
  6. システム環境変数のPathを開く
  7. 新規をクリック
  8. 解凍先のパスを入力
  9. OKをクリック

Hugoで新しいサイトを作る

サイトの基礎を作る

  1. コマンドプロンプトを開く
  2. 解答した場所に作業ディレクトリを移す
  3. hugo new site サイト名を実行

テーマをインストールする

  1. git initを実行
  2. git submodule add テーマのGithubリポジトリURL themes/テーマ名
  3. config.tomlにtheme = "テーマ名"を書き込む

このときhasCJKLanguage = trueとついでに書いておくと
記事一覧を表示したときに、記事の冒頭70語だけを表示してくれる。

テーマが正常に動作しているかどうかを確かめる

  1. hugo start -Dを実行
  2. http://localhost:1313/にアクセス

Github上で動作することを想定してconfigを変更する

  1. config.tomlのbaseURLを https://ユーザーID.github.io/リポジトリ名/ に変更する

ビルド

  1. hugo -Dを実行

Githubにアップロードする(初回)

  1. cd publicを実行
  2. git initを実行
  3. git add .を実行
  4. git commit -m コメントを実行
  5. git remote add origin https://github.com/ユーザーID/リポジトリ名.gitを実行
  6. git push -u origin masterを実行
  7. 別ウィンドウでユーザーIDとパスワードの入力を求められるので入力

Github Pagesの公開設定する

  1. リポジトリにアクセス(https://github.com/ユーザーID/リポジトリ名)
  2. Settingsを開く
  3. 下にスクロールしてGithub Pagesの設定項目を見つける
  4. Sourceにmasterを選択
  5. セーブを押す

記事を作る

  1. hugo new posts/Title.mdを実行
    (ファイル名に日本語を指定すると文字化けするので英語で指定した後手動で日本語のファイル名を指定する必要がある。)
  2. マークダウンで書く (次回の記事はこれについて書くかも？)

Githubにアップロードする(二回目以降)

  1. publicの中身を削除 (これはどちらでも)
  2. コマンドプロンプトを開く
  3. 解答した場所に作業ディレクトリを移す
  4. hugo -Dを実行
  5. cd publicを実行
  6. git add .を実行
  7. git commit -m コメントを実行
  8. git push -u origin masterを実行
    (分かる人はこの一連の動作をbatファイルに書くと楽)
